## 작업 원칙

- 기준 원문은 `index.html`의 한글판이다.
- `ja.html`, `en.html`은 한글판의 HTML 구조, 이미지, 영상, 페이지 라우팅, 앵커를 동일하게 유지한다.
- 기존 일본어·영어 파일의 낡은 문구를 참고하지 않는다.
- 한글 원문을 직접 번역한다.
- 한 페이지의 본문·버튼·이미지 대체텍스트·메뉴가 모두 번역되고 확인된 뒤에만 완료로 표시한다.
- 일본어 전체 완료 후 영어 전체를 완료한다.
- 두 언어가 모두 끝난 뒤에만 전체 검수, 커밋, 푸시를 한다.

## 현재 상태

- `ja.html`, `en.html`을 `index.html`의 최신 구조로 다시 생성했다.
- 외부 번역 서버와 자동 번역 API는 사용하지 않았다.
- 한글 원문을 기준으로 일본어와 영어를 직접 번역했다.
- 본문, 메뉴, 버튼, 이미지 대체텍스트, 동적 안내문, 메일 제목까지 번역했다.
- 두 언어 파일의 한글 잔여는 각각 0개다.

## 완료 기준

1. 각 언어 파일의 메뉴, 본문, 버튼, 대체텍스트가 해당 언어로 표시된다.
2. `index.html`, `ja.html`, `en.html`의 섹션 수와 `data-page` 구조가 동일하다.
3. 언어 전환 링크는 `KR → index.html`, `EN → en.html`, `JP → ja.html`로 상호 이동한다.
4. 이미지와 영상 경로는 한글판과 동일하다.
5. 모바일과 데스크톱에서 페이지 라우팅과 메뉴가 유지된다.
6. 모든 항목이 완료된 뒤에만 한 번 커밋하고 `main`으로 푸시한다.
