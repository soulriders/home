# SDA 홈페이지 문장 전략 및 콘텐츠 위계

작성일: 2026-07-08  
목적: Receivepower 홈페이지의 문장 강도, 주장 방식, 콘텐츠 배치 순서를 고정한다.  
적용 범위: 한국어 홈페이지 원고, 서브페이지 원고, IR/기술/IP/제품/시장/비즈니스 문장.

---

## 1. 핵심 원칙

Receivepower 홈페이지의 문장은 약하게 피하지 않는다.  
특허와 기술 문서로 뒷받침되는 구조적 우위는 강하게 쓴다.

다만 강한 문장은 감정적 과장이 아니라 구조, 특허, 인증 요구, 비용 효과, 운항 효과에 붙인다.

```text
강한 주장 = 기술 구조 + 특허 축 + 인증 논리 + 비용 효과 + 운항 효과
```

홈페이지의 핵심 문장은 다음 축을 벗어나지 않는다.

```text
SDA는 조종면 중심 항공기 구조를 소프트웨어 정의 비행체 구조로 바꾼다.
Lambda는 SDA를 실제 기체와 시장에서 증명하는 모델이다.
화물 회랑은 Lambda의 정체성이 아니라 첫 적용 시장이다.
IP는 단순 보호 장치가 아니라 비용 구조와 안전 구조를 지배하는 핵심 방어 자산이다.
```

---

## 2. 문장 톤

### 2.1 기본 톤

사용한다.

- 절대우위
- 지배한다
- 장악한다
- 잠근다
- 정면으로 겨냥한다
- 직접 경쟁한다
- 구조를 결정한다
- 인증 논리를 완성한다
- 시장을 연다
- 비용 구조를 바꾼다
- 안전 구조를 내장한다

피한다.

- 가능성이 있습니다
- 연결합니다
- 지향합니다
- 기대됩니다
- 도움이 됩니다
- 설명할 수 있습니다
- 기여합니다
- 참고 근거입니다
- 검토합니다

예외: 아직 확정되지 않은 제품 라인업, 투자 조건, 해외 규제 대응, 개인 투자 모집 방식은 단정하지 않는다.

### 2.2 약한 문장과 강한 문장

| 약한 문장 | 강한 문장 |
| --- | --- |
| 기술들이 하나로 연결됩니다. | 5대 기술 해자가 하나의 SDA 비행 아키텍처를 성립시킵니다. |
| 인증 설명이 가능해집니다. | 결정론적 비행 거동으로 인증기관에 제출 가능한 제어 논리를 만듭니다. |
| 비용 절감에 기여합니다. | 조종면 제거가 생산 공정, 정비 항목, 인증 항목을 동시에 줄입니다. |
| 트럭과 경쟁할 수 있습니다. | 트럭 운임과 직접 경쟁하는 비행체입니다. |
| 안전성을 높입니다. | 동력 상실 상황에서도 제어하강을 수행하도록 설계되었습니다. |
| 시장 진입 가능성이 있습니다. | 화물 운항 데이터가 유인 확장과 인증 경로를 여는 실증 자산이 됩니다. |

## 3. 금지 표현

다음 표현은 홈페이지에서 사용하지 않는다.

- `~가 아닙니다`
- `~은 아닙니다`
- `공개 표현`
- `홈페이지에서는`
- `별도 검토 자료`
- `보조 근거`
- `내부 메모`
- `방문자 시퀀스`
- `우선권 주장`
- `가능성이 있습니다`의 반복
- `관계자본`처럼 의미가 모호한 단어

특히 다음 문장 구조는 피한다.

```text
A가 아닙니다. B입니다.
```

대신 바로 B를 말한다.

```text
SDA는 비행모드를 소프트웨어로 지배하는 VTOL 항공 플랫폼입니다.
```

## 4. 콘텐츠 위계

홈페이지 전체의 읽기 순서는 아래 순서를 따른다.

```text
1. 정체성
2. 비행 구조
3. IP 절대우위
4. 첫 적용 시장
5. 제품/체급
6. 사업 모델
7. 인증/안전 구조
8. Physical AI 확장
9. 협력 대상
```

이 순서가 무너지면 Lambda가 화물기, 드론, 전세기, 항공로봇 중 하나로 좁게 읽힌다.  
Lambda의 상위 정체성은 SDA 기반 소프트웨어 정의 비행체다.

## 5. Home 위계

### 5.1 Hero

역할: 5초 안에 정체성을 고정한다.

우선순위:

```text
SDA / 소프트웨어 정의 비행체
조종면 제거
비동기 틸팅
가상조종면
이산 MMA
무동력 고양력 제어
트럭 운임과 직접 경쟁
```

Hero에서 피할 것:

- 화물기처럼 보이게 시작
- 드론처럼 보이게 시작
- 연구 과제처럼 보이게 시작
- 인증 조항을 첫 문장에 노출
- 제품 라인업을 먼저 설명

권장 구조:

```text
절대우위의 소프트웨어 정의 비행체.
Lambda는 조종면을 제거하고, 비동기 틸팅·가상조종면·이산 MMA·무동력 고양력·차동항력 제어로 비행모드를 지배하는 SDA 플랫폼입니다.
첫 적용 시장은 트럭 운임과 직접 경쟁하는 지역 간 물류 회랑입니다.
```

### 5.2 비행 이미지 섹션

역할: 말보다 먼저 기체 구조를 보여준다.

포함:

- 수직이착륙
- 전환
- 순항
- 제어하강
- 독립 틸팅 로터
- 조종면 없는 구조

문장 방향:

```text
SDA는 수직이착륙, 전환, 순항, 제어하강을 하나의 기체 구조 안에서 지배합니다.
```

### 5.3 기술 축

역할: SDA의 절대우위를 3개 축으로 압축한다.

권장 축:

```text
조종면을 제거한 비행체
비행모드 전환 아키텍처
무동력 고양력·차동항력 활강
```

각 축의 문장 구조:

```text
구조 변화 → 비용 효과 → 인증/운항 효과
```

## 6. IP 위계

IP 페이지는 제품 소개 페이지가 아니다.  
기체보다 큰 권리 구조와 산업 표준 가능성을 보여주는 페이지다.

### 6.1 IP 페이지의 핵심 문장

```text
Lambda IP는 운항 비용과 안전 구조를 동시에 장악하는 절대우위입니다.
```

### 6.2 5대 핵심 기술

5대 핵심 기술은 병렬 나열이 아니다.  
각 기술은 SDA를 성립시키는 필수 축이다.

```text
1. 비동기 틸팅 추력기관
2. Stagger 기반 가상조종면
3. 이산 매트릭스 기반 비행 관리
4. 무동력 고양력 제어
5. SADS 기반 가상피토관
```

### 6.3 기술별 역할

| 기술 | 홈페이지 의미 | 강한 표현 |
| --- | --- | --- |
| 비동기 틸팅 | VTOL 전환 위험을 단계 제어로 분산 | 전환비행의 병목을 구조적으로 제거 |
| 가상조종면 | 물리 조종면 제거 | 생산비, 정비비, 인증 항목을 동시에 줄임 |
| 이산 MMA | 상태별 제어 모드 고정 | 블랙박스 자율성을 배제하고 결정론적 비행을 구현 |
| 무동력 고양력·차동항력 | 동력 상실 대응 | PL.2105(g)를 정면으로 겨냥한 제어하강 구조 |
| SADS 가상피토관 | 상태 추정과 모드 전환 트리거 | 센서융합으로 Flight OS의 상태 인식 기반을 만듦 |

### 6.4 IP 문장 금지

약하다.

```text
5대 기술이 연결됩니다.
안전성에 연결됩니다.
운항 효과로 이어집니다.
```

강하다.

```text
5대 기술 해자가 하나의 SDA 비행 아키텍처를 성립시킵니다.
각 기술 해자는 기체 가격, 정비비, 운항률, 안전성의 구조를 직접 결정합니다.
틸팅 전환, 이산 제어, 무동력 고양력, 차동항력, SADS 상태 추정이 하나의 인증 논리를 완성합니다.
```

## 7. 제품 위계

제품 페이지는 Lambda를 화물기 하나로 고정하지 않는다.

기본 위계:

```text
SDA 플랫폼
Lambda 모델
검증 체급
화물 운항 체급
유인 확장 체급
특수 임무 확장
```

문장 방향:

```text
Lambda는 화물 회랑에서 시장을 여는 소프트웨어 정의 비행체입니다.
```

화물은 첫 적용 시장이다.  
Lambda의 본질은 소프트웨어 정의 비행체다.

## 8. 시장 위계

시장 페이지의 목적은 시장 규모를 넓게 보이게 하는 것이 아니다.  
첫 적용 시장의 필연성을 보여주는 것이다.

핵심 문장:

```text
첫 적용 시장은 트럭 운임과 직접 경쟁하는 지역 간 물류 회랑입니다.
```

시장 설명 순서:

```text
도로가 늦은 구간
해상이 일정에 묶이는 구간
기존 항공기가 비싼 구간
100~500kg급 시간 민감 화물
반복 운항 데이터
유인 확장과 인증 경로
```

## 9. 비즈니스 위계

비즈니스 페이지는 돈 이야기부터 시작하지 않는다.  
SDA가 지적자산, Flight OS, 제조 파트너십, 운항 데이터로 확산되는 구조를 먼저 말한다.

권장 순서:

```text
IP 통제
Flight OS
기체/리스
회랑 운항
MRO
감항유지 데이터
지역 JV
```

강한 문장:

```text
SDA의 목표는 하드웨어 제조사가 아니라 항공 물류의 원가 구조와 운항 표준을 장악하는 것입니다.
```

가격 문장 배치:

```text
지역 실증 연구용역 약 2,000만원
전용기체 개발/제작 약 2억원
기체 1억원 목표
라이프사이클 매출 50억원
```

가격은 맨 위가 아니라 상세 카드나 하단 설명에 둔다.

## 10. IR 위계

IR 페이지는 기술 낭만보다 자본 논리를 먼저 보여준다.

순서:

```text
반복수익 구조
인증 스케줄의 투명성
기체 원가 절감
운항 데이터
MRO / Flight OS / 감항유지 데이터
Exit 가능성
```

인증 메시지:

```text
SDA는 PL.2105(g)를 정면으로 겨냥해 만든 비행체입니다.
```

무조종면과 무동력 활강을 함께 말할 때는 차동항력을 반드시 붙인다.  
조종면이 없는데 동력도 없으면 어떻게 조향하는가라는 반론을 막기 위한 핵심 설명이다.

```text
동력 완전 상실 시에도 로터의 자동회전과 차동항력이 가상의 조종면 역할을 수행합니다.
SDA는 조종면 없는 기체에서도 무동력 상태의 양력, 자동회전, 차동항력으로 제어하강 논리를 세웁니다.
```

단, 실제 인증 완료처럼 보이는 문장은 쓰지 않는다.

금지:

```text
인증을 통과했습니다.
인증이 끝났습니다.
규제가 해결됐습니다.
```

허용:

```text
인증 요구를 정면으로 겨냥했습니다.
인증기관에 제출 가능한 결정론적 비행 구조를 만듭니다.
인증 논리를 완성합니다.
```

## 11. Company 위계

Company는 단순 항공기 회사 소개가 아니다.  
리시브파워가 Physical AI와 실제 기계를 다루는 회사라는 상위 정체성을 보여준다.

순서:

```text
2019년부터 지능형 로봇 개발
Humanoid부터 Aerial robot까지
2020년 비동기 틸트로터 특허 출원
현재 권리 획득
2026년 SDA 아키텍처 확장
트럭 운임과 경쟁하는 비행체
```

강한 문장:

```text
Lambda SDA는 자동차 수준의 비용 구조와 소프트웨어 정의 비행을 결합한 VTOL 항공 플랫폼입니다.
```

## 12. Physical AI 위계

Physical AI는 홈페이지 초반에 두지 않는다.  
SDA 정체성과 IP 우위를 먼저 설명한 뒤, 회사의 확장성으로 배치한다.

역할:

```text
SDA는 첫 증명이다.
Receivepower는 실제 기계를 움직이는 Physical AI 회사다.
비행체 제어에서 얻은 상태 인식, 센서융합, 실패 대응, 제어 루프는 휴머노이드와 산업 로봇으로 확장된다.
```

## 13. 방문자별 연결 위계

방문자 대상은 메인페이지에서 내부 기획 용어처럼 보이지 않게 다룬다.

표현:

```text
창조적인 협력
```

대상별 연결:

| 대상 | 핵심 관심 | 연결 페이지 |
| --- | --- | --- |
| 자본투자 | 반복수익, 인증 스케줄, Exit | IR |
| 기술협력 | 비동기 틸팅, Flight OS, 전용기체 개발 | 기술 / IP |
| 지방자치단체(한국) | UAM 도시 시뮬레이션, 기체 실증, 노선 실증 | 시장 / 비즈니스 |
| 지방자치단체(일본) | 공장, MRO, 지역 회랑 | 시장 / 비즈니스 |
| 드론·비행체 기업 | 100~500kg급 체급 확장 | 기술 / 제품 |
| 휴머노이드·지능형로봇 기업 | 물류 연동 DX 솔루션 | 기술 / 시장 |

## 14. 최종 체크리스트

홈페이지 원고 수정 전후로 반드시 확인한다.

- [ ] Lambda가 화물기로만 읽히지 않는가
- [ ] SDA가 상위 플랫폼으로 먼저 보이는가
- [ ] 화물 회랑은 첫 적용 시장으로 배치됐는가
- [ ] IP가 제품 스펙보다 큰 권리 구조로 보이는가
- [ ] 5대 기술이 병렬 나열이 아니라 SDA 아키텍처를 성립시키는 구조로 보이는가
- [ ] 인증 문장은 완료처럼 보이지 않고, 요구를 정면으로 겨냥한 구조로 보이는가
- [ ] 약한 표현 `가능성`, `지향`, `연결`, `기여`가 핵심 문장에 남아 있지 않은가
- [ ] 숫자와 비용은 과장 대신 구조적 근거로 붙어 있는가
- [ ] Company는 Physical AI 회사의 방향을 보여주는가
- [ ] Business는 단순 기체 판매가 아니라 IP, Flight OS, MRO, 데이터, JV 구조를 보여주는가
- [ ] IR은 반복수익과 인증 스케줄을 먼저 보여주는가
